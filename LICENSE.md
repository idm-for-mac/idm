## 🛡️ Tips for Maximum Download Speed

To get the fastest possible downloads with IDM for Mac:

1. **Increase connection count per download** — Go to Preferences → Download → Max connections. Default is 16. Increase to 32 if your internet plan supports 500+ Mbps.

2. **Whitelist IDM in your firewall/antivirus** — Some security tools throttle multi-connection downloads. Add IDM to the exception list.

3. **Use a download folder on an SSD** — Mechanical hard drives (especially older iMacs) can bottleneck multi-threaded writes. Set your default download folder to an SSD volume.

4. **Enable "Speed Limiter" for background downloads** — Counterintuitively, throttling IDM to 80% of your bandwidth while gaming or streaming prevents bufferbloat and actually improves overall perception.

5. **Test different mirrors/sources** — Some servers impose per-connection limits. For slow sources, reduce connections to 4 or 8 instead of increasing.

6. **Use LAN/Ethernet when possible** — Wi-Fi interference and airtime fairness can limit multi-threaded performance. Ethernet eliminates this.

## 🔧 Common Issues and Fixes

### ❌ "IDM for Mac doesn't capture downloads in Safari"
**Fix:** Safari requires explicit permission for extensions. Go to **Safari → Settings → Extensions → Enable 'IDM for Mac'**. Then ensure "Can access" is set to "All websites" (or your specific allowed sites). Restart Safari after enabling.

---

### ❌ "My 32-part download is actually slower than 8 parts"
**Fix:** Some CDNs and older servers rate-limit per connection. Reducing simultaneous connections can actually increase total throughput. Try **Preferences → Download → Max connections** set to 8 or 4. Also test at off-peak hours.

---

### ❌ "Download keeps failing with 'Network Error' halfway through"
**Fix:** This usually indicates an unstable Wi-Fi connection or a server timeout. Enable **Preferences → Download → Auto-resume after failure** (it's on by default). If the issue persists, try using a VPN (some ISPs throttle multi-part downloads) or switch to Ethernet.

---

### ❌ "Video sniffer doesn't work on YouTube/Twitch"
**Fix:** Streaming platforms frequently change their video delivery formats. After a platform update, it may take 24–48 hours for IDM's sniffer rules to update. Go to **Help → Update Sniffer Rules** to manually fetch the latest patterns. If still broken, update to the latest IDM version.

---

### ❌ "IDM for Mac uses too much CPU during downloads"
**Fix:** Multi-part downloads do use more CPU than single-threaded ones. On Intel Macs, reduce the connection count to 8 or 12. On Apple Silicon, this is rarely an issue. Also check **Preferences → Performance → Background priority** to lower CPU usage when other apps are active.

---

### ❌ "The browser extension keeps disabling itself (Chrome)"
**Fix:** Chrome sometimes disables extensions that it perceives as slowing down browsing. Go to **chrome://extensions** → find IDM for Mac → enable "Allow in incognito" (oddly, this stabilizes the extension). Also ensure Chrome is up to date.

---

### ❌ "Can I use IDM for Mac with VPN or Proxy?"
**Yes!** Go to **Preferences → Network → Proxy Settings**. IDM supports HTTP, HTTPS, SOCKS4, and SOCKS5 proxies. For VPNs, if the VPN is system-wide, IDM automatically uses it without additional configuration.

## ❓ FAQ

**Q1: Is IDM for Mac the same as the Windows IDM?**
> No — and that's a good thing. **IDM for Mac** was built from scratch for macOS using Swift and Apple's networking APIs. It shares the same core concepts (multi-threaded downloads, browser integration, video sniffing) but the implementation is fully native. Windows IDM will not run on Mac, and IDM for Mac will not run on Windows.

**Q2: Does IDM for Mac have a free version?**

---

**Q3: How do I uninstall IDM for Mac completely?**
> Drag `IDM for Mac.app` from `/Applications` to the Trash. Then manually remove the following folders if you want to delete settings:
> - `~/Library/Application Support/IDM for Mac/`
> - `~/Library/Preferences/com.idm.mac.plist`
> Browser extensions must be removed manually from each browser's extension manager.

---

**Q4: Does IDM for Mac work with M1/M2/M3/M4 Macs?**
> **Yes — perfectly.** IDM for Mac is a **universal binary** with native ARM64 code for Apple Silicon and x86_64 for Intel. On M-series Macs, it runs natively (not under Rosetta) and takes full advantage of the efficiency cores for background downloads.

---

**Q5: Is there a portable version (run from USB drive)?**
> Not officially. IDM for Mac requires installation to `/Applications` for browser extension integration to work properly. Running from a USB drive would break the extension communication layer.

---

**Q6: How is IDM for Mac different from Downie or Folx?**
> Downie is specialized for video downloads (excellent at that) but lacks general file download features. Folx is a general downloader but is aging (no native Apple Silicon version, outdated UI). IDM for Mac combines Downie's video capabilities with Folx's general download features — and adds proper Safari integration, which neither does well.

---

**Q7: Does IDM for Mac support downloading from Google Drive, Dropbox, etc.?**
> **Partially.** IDM can capture the final download link from these services after you click the "Download" button in your browser. However, it cannot bypass waiting times or captchas on the landing pages. For large Google Drive files that trigger virus scans, IDM will wait until the actual file link becomes available.

---

**Q8: Will IDM for Mac slow down my browsing?**
> No. The browser extensions only activate when you click a download link or right-click a specific element. They do not scan pages in the background, inject ads, or send your browsing data anywhere. In testing, performance impact is under 2% CPU / 10 MB RAM.
